## Project Overview

This is the official corporate website for **Controva LLC**, an engineering-driven digital growth agency. It is a highly dynamic, premium Single Page Application (SPA) built with React, TypeScript, and Vite. The site is designed for a B2B audience and emphasizes performance, animation, and conversion optimization.

**Key characteristics:**
- **SPA with client-side routing** via React Router DOM v7.
- **Premium animations** using Framer Motion (blur-reveal typography, mouse-following spotlight, page transitions, scroll-triggered reveals).
- **Dynamic theming** with 10 built-in color palettes plus custom color picker, dark/light mode, and persistent user preferences via `localStorage`.
- **Blog system** with Markdown rendering (`react-markdown` + `remark-gfm`), real-time discussion/comments via Supabase OAuth, and a password-protected admin panel (`/admin/blog`).
- **SEO-first architecture** with auto-generated `sitemap.xml`, `robots.txt`, `rss.xml`, JSON-LD structured data, Open Graph, and Twitter Card meta tags.
- **Deployment target:** Apache shared hosting (Hostinger) with `.htaccess` rewrite rules; also configured for Vercel.

**Domain:** `https://www.controvallc.com`

## Build and Development Commands

All commands run from the project root.

```bash
# Install dependencies
npm install

# Start development server (runs on http://localhost:3000)
npm run dev

# Type-check and build for production
npm run build

# Preview production build locally
npm run preview

# Lint the codebase
npm run lint
```

**Build output:** The `dist/` directory is generated by `vite build`. It contains static assets ready for deployment.

**Important:** The `build` script runs `tsc -b && vite build`. TypeScript errors will block the build.

## Code Style Guidelines

### TypeScript
- **Strict mode enabled** (`strict: true` in `tsconfig.app.json`).
- **No unused locals or parameters** (`noUnusedLocals`, `noUnusedParameters`).
- **Module resolution:** `bundler`. Imports use `.ts` / `.tsx` extensions implicitly.
- **Path alias:** `@/` maps to `./src/`. Always use `@/components/...`, `@/data/...`, etc.
- **React JSX transform:** `react-jsx` (no `import React` needed).

### Styling Conventions
- **Tailwind CSS** is used for utility-based layout.
- **Custom CSS properties (variables)** drive the dynamic theming system. All color values that should respond to theme changes MUST use CSS vars, not hardcoded Tailwind classes.
- **Key CSS variables** (defined in `index.css` and updated by `ThemeContext`):
  - `--bg-primary`, `--bg-secondary`, `--bg-tertiary`, `--bg-card`
  - `--text-primary`, `--text-secondary`, `--text-muted`
  - `--border-subtle`, `--border-active`
  - `--accent-1`, `--accent-2`, `--accent-gradient`, `--accent-glow`, `--accent-subtle`, `--accent-border`
  - `--glow-tl`, `--glow-tr`, `--glow-bl`, `--glow-br`
- **Font families:**
  - Headings / display: `font-family: 'Space Grotesk', ...` (use `.font-display` utility)
  - Body text: `font-family: 'Inter', ...` (use `.font-body` utility)
- **Golden Ratio design system** (`φ = 1.618`) is used for font sizes (`--fs-*`), spacing (`--sp-*`), and border radii. Prefer these custom properties over arbitrary Tailwind values when possible.
- **Component-level hover effects** are often implemented via inline `style` + `onMouseEnter`/`onMouseLeave` to dynamically respond to the current theme, rather than static Tailwind classes.

### Component Patterns
- **Framer Motion** is the primary animation library. Use `motion.div`, `initial`, `animate`, `whileInView`, `viewport={{ once: true }}` for scroll-triggered reveals.
- **Reusable components:**
  - `SEO.tsx` — Wraps `react-helmet-async` for page-specific meta tags and JSON-LD schema.
  - `SectionHeader.tsx` — Standard label + headline block with scroll-reveal animation.
  - `ScrollReveal.tsx` — Generic scroll-triggered fade/slide wrapper.
  - `GradientButton.tsx` / `BorderButton.tsx` — Styled CTA buttons with hover animations.
- **Context providers:** `ThemeProvider` and `BlogProvider` wrap the entire app in `App.tsx`.

### File Naming
- Components: PascalCase (`HeroSection.tsx`, `BlogContext.tsx`)
- Utilities / data: camelCase (`content.ts`, `supabase.ts`)
- Directories: kebab-case for multi-word (`case-studies/`, `blog-discussion/`)

## Routing

React Router DOM v7 (`BrowserRouter`) handles client-side navigation.

| Route | Page Component | Notes |
|-------|---------------|-------|
| `/` | `HomePage` | Landing page with hero, bento grid, testimonials |
| `/services` | `ServicesPage` | Service cards, process, detailed breakdown, pricing |
| `/case-studies` | `CaseStudiesPage` | Portfolio grid with metrics |
| `/about` | `AboutPage` | Mission, values, team |
| `/blog` | `BlogPage` | Blog listing with featured post |
| `/blog/:slug` | `BlogSlugPage` | Individual article with Markdown + discussion |
| `/admin/blog` | `BlogAdminPage` | **Password-protected** (`controva2026`) admin panel |
| `/contact` | `ContactPage` | Contact form + info |

**Scroll behavior:** `ScrollToTop` component resets `window.scrollTo(0, 0)` on every route change.

**Page transitions:** `AnimatePresence` with `mode="wait"` wraps all routes. Each page is wrapped in a `motion.div` that fades in/out.

## Backend / Data

### Supabase
The project uses Supabase for:
1. **Blog posts** (`blog_posts` table) — CRUD operations in `BlogContext.tsx`.
2. **Comments** (`comments` table) — Real-time discussion on blog articles via `BlogDiscussionSection.tsx`.
3. **OAuth authentication** — GitHub, Google, and LinkedIn OIDC sign-in for commenting.

**Schema assumptions:**
- `blog_posts` table with columns matching the `BlogPost` interface (id, title, excerpt, category, author, date, readTime, slug, image, content).
- `comments` table with columns: id, post_slug, user_id, user_name, user_avatar, content, created_at, status.

**Fallback behavior:** If Supabase is unavailable or tables are empty, the blog falls back to hardcoded `seedPosts` from `src/data/blog.ts`.

### Admin Panel
- Route: `/admin/blog`
- Password: `controva2026` (hardcoded in `BlogAdminPage.tsx`)
- Authentication state is stored in `sessionStorage` under key `blog_admin`.
- Seed posts (IDs like `'1'`, `'2'`) cannot be edited or deleted via the admin panel — only Supabase-backed posts with UUIDs can.

## SEO & Deployment

### SEO Features
- **Dynamic `<SEO>` component** on every page sets title, description, Open Graph, Twitter Card, canonical URL, and JSON-LD structured data.
- **Static files in `public/`:**
  - `sitemap.xml` — Lists core pages with priorities and lastmod.
  - `robots.txt` — Allows all crawlers including AI bots (GPTBot, ClaudeBot, PerplexityBot, etc.). Blocks `/api/`, `/admin/`, `/*.json$`.
  - `rss.xml` — RSS 2.0 feed for blog syndication.
- **JSON-LD schemas:** `WebSite` (in `index.html`), `ProfessionalService` (default in `SEO.tsx`), and `Article` (on blog slug pages).

### Deployment

**Apache / Hostinger (Primary):**
- The `public/.htaccess` file rewrites all non-file/directory requests to `index.html`, enabling SPA routing.
- Upload the contents of the `dist/` directory to the Apache document root.

**Vercel (Secondary):**
- `vercel.json` configures SPA rewrites and sets the output directory to `dist`.
- Build command: `npm run build`

## Security Considerations

- **Admin password is hardcoded** (`controva2026` in `BlogAdminPage.tsx`). This is a simple client-side gate, not a true security boundary. Do not store sensitive data behind it.
- **Supabase anon key is exposed** in the client bundle (standard for Supabase client-side apps). Row Level Security (RLS) should be enabled on Supabase tables to prevent unauthorized data access.
- **No CSP headers** are configured in the build output. If deploying to a platform that supports custom headers, consider adding a Content-Security-Policy.
- **Contact form** is currently a mock (`alert()` on submit). It does not send data anywhere. If wiring it up, use a backend endpoint or a form service (e.g., Formspree, Getform).

## Common Tasks for Agents

### Adding a new page
1. Create the page component in `src/pages/`.
2. Add the route in `src/App.tsx` inside `AnimatedRoutes`.
3. Add an SEO `<SEO>` component to the new page with appropriate title/description.
4. Update `public/sitemap.xml` if the page should be indexed.

### Adding a new theme palette
1. Open `src/context/ThemeContext.tsx`.
2. Add a new object to `defaultPalettes` array: `{ name: '...', accent1: '#...', accent2: '#...' }`.
3. The new palette will automatically appear in the `ThemeCustomizer` preset grid.

### Adding a new blog post (manually)
1. Add a new entry to the `blogPosts` array in `src/data/blog.ts`.
2. Ensure the `slug` is unique and URL-friendly.
3. Optionally add the post to `public/rss.xml`.

### Adding a new section to the homepage
1. Create the section component in `src/components/sections/home/`.
2. Import and render it in `src/pages/HomePage.tsx`.

### Modifying the navbar links
- Edit the `navLinks` array in `src/data/content.ts`.
- The navbar auto-generates links from this array (excluding `/contact`, which is handled specially).
